# Epics and Stories

## Epic 1: Member and Household Management
### Story 1.1: Create and update member records
As an administrator, I want to create and update member records so that the church directory stays accurate.

Acceptance Criteria:
- An administrator can create a new member record.
- Core fields such as name, contact info, household, and status are stored.
- Existing records can be edited and updated.

### Story 1.2: Assign members to divisions
As a church administrator, I want to assign members to divisions such as Sunday School, Youth, and Adult classes so that ministry groups are organized.

Acceptance Criteria:
- A member can be assigned to one or more divisions.
- Divisions are stored as identifiable group types.
- The assigned division is visible on the member profile.

### Story 1.3: Search and filter the directory
As a staff member, I want to search and filter members so that I can find people quickly.

Acceptance Criteria:
- Users can search by name or household.
- Users can filter by status, ministry tag, division, or active/inactive state.

## Epic 2: Attendance and Event Tracking
### Story 2.1: Record attendance for services
As a ministry leader, I want to record attendance so that participation can be tracked over time.

Acceptance Criteria:
- Attendance can be recorded for a service or event.
- Attendance entries are associated with a date and person.
- Attendance history can be viewed for a person or group.

### Story 2.2: Track attendance by division
As a class leader, I want to track attendance by division so that Sunday School, Youth, and Adult class participation is visible.

Acceptance Criteria:
- Attendance can be recorded for a specific division.
- Division attendance summaries are visible by date and class.
- Leaders can compare attendance across divisions.

### Story 2.2: Manage event RSVPs
As an event coordinator, I want to track RSVPs so that I can plan events more effectively.

Acceptance Criteria:
- Events can be created with a date, location, and description.
- People can be marked as attending or not attending.
- Event attendee counts are visible.

## Epic 3: Giving and Stewardship
### Story 3.1: Record donations
As an administrator, I want to record donations so that giving history is maintained.

Acceptance Criteria:
- Donations can be entered for a person or household.
- Donation details include amount, date, and type.
- Donation records can be viewed later.

### Story 3.2: Track building fundraising campaigns
As a church leader, I want to track fundraising campaigns for the church building so that progress toward completion is visible.

Acceptance Criteria:
- A fundraising campaign can be created for the building project.
- Pledges and contributions can be recorded against the campaign.
- Progress toward the campaign target is visible in summary reports.

### Story 3.3: Review giving summaries
As a church leader, I want to review giving summaries so that I can monitor stewardship trends.

Acceptance Criteria:
- A basic summary view shows totals by period or donor.
- Reports can be exported or reviewed by staff.
- Fundraising progress can be viewed alongside general giving summaries.

## Epic 4: Communication and Follow-Up
### Story 4.1: Send announcements
As a staff member, I want to send announcements so that members receive timely updates.

Acceptance Criteria:
- A message can be composed and sent to a selected group.
- The audience can be filtered by tag, ministry, or household.
- Sent communications are logged.
